Want to test the performance and resource usage of our code with different
combinations of the following:
	Nodes: 1, 2, 4, 8
	Cores: 1, 2, 4, 8, 16, 32
	Memory per core: 64MB, 128MB, 512MB, 1GB, 1.5GB, 3GB

We only want to run our tests jobs on the "mole" class of nodes.

We want to show how performance differs across multiple machines vs a single
machine and using different memory "budgets"
(persumably budget = nodes * ntask-per-node * mem-per-cpu
where
	nodes = number_of_nodes,
	ntask-per-node = cores_per_node = number_of_cores,
and
	mem-per-cpu = memory_per_core).

In order to give us room to vary the various parameters, we may want to fix our
first budget at a middle value in each of the three. So that's
	nodes = 2
	cores_per_node = 8
	mem_per_core = 512MB
=> budget = 8192MB

To get combinations, we can trade 1 for another. Good combinations might be:
{(nodes_0, cores_per_node_0, mem_per_core_0), ...} = {
	(1, 16, 512MB),
	(2, 4, 1GB),
	(2, 8, 512MB),  // As above
	(2, 32, 128MB),
	(4, 4, 512MB),
	(4, 16, 128MB),
	(8, 8, 128MB),
	(8, 2, 512MB),
}
