Want to test the performance and resource usage of our code with different
combinations of the following:
	Nodes: 1, 2, 4, 8
	Cores: 1, 2, 4, 8, 16, 32
	Memory per core: 64MB, 128MB, 512MB, 1GB, 1.5GB, 3GB

We only want to run our tests jobs on the "mole" class of nodes.

We want to show how performance differs across multiple machines vs a single
machine and using different memory "budgets"
(persumably budget = nodes * ntask-per-node * mem-per-cpu
where
	nodes = number_of_nodes,
	ntask-per-node = cores_per_node = number_of_cores,
and
	mem-per-cpu = memory_per_core).

In order to give us room to vary the various parameters, we may want to fix our
first budget at a middle value in each of the three. So that's
	nodes = 2
	cores_per_node = 8
	mem_per_core = 512MB
=> budget = 8192MB

To get combinations, we can trade 1 for another. Good combinations might be:
{(nodes_0, cores_per_node_0, mem_per_core_0), ...} = {
	(1, 16, 512MB),
	(2, 4, 1GB),
	(2, 8, 512MB),  // As above
	(2, 32, 128MB),
	(4, 4, 512MB),
	(4, 16, 128MB),
	(8, 8, 128MB),
	(8, 2, 512MB),
}

Submit a job under Slurm using `sbatch`




We are testing the MPI implementation performance by testing 3 different cases with 3 different memory budgets. 

Maximum Nodes: Using as many nodes as possible, distributing the workload across the most machines possible. This tests the overhead and benefit of the inter-node MPI communicaiton.

Maximize Cores Per Node: Concentrate all cores on as few nodes as possible, maximizing intra-node parallelism and shared memory bandwidth

Maximize Memory per Core: Allocate the largest possible memory per core, using fewer total ranks. This tests whether memory-bound workloads benefit from reduced core density.

Budget A: ~4096 MB:
  1. Maximizing Nodes
    - Nodes: 4
    - Cores per Node: 4
	- Memory per Core: 512 MB

  2. Maximizing cores per node
    - Nodes: 1
    - Cores per Node: 16
	- Memory per Core: 256 MB

  3. Maximizing memory per core
